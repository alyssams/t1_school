Why do we want to animate elements?

The CSS animations feature lets us create smooth, multistep animations.

An animation lets an element gradually change from one style to another.

  • You can change as many properties you want, as many times you want.
  • We can then control their direction, play state, how they iterate, and what happens before and after they execute with a few simple CSS declarations.
  • Animations only execute when we bind them to a selector,and the changes in CSS values are defined separately in a set of keyframes.
  • You can specify when the change will happen in percent, or you can use the keywords "from" and "to" (which represents 0% and 100%).
  • 0% represents the start of the animation, 100% is when the animation is complete.

Browser Support

Keyframes

The @keyframes rule is where the animation is created.

Specify a CSS style inside the @keyframes rule and the animation will gradually change from the current style to the new style.


		.box {
			width:100px;
			height:100px;
			background:blue;
			animation-name: blue-box;
			animation-duration: 5s;
		}
		@keyframes blue-box {
			from { background:   blue; }
			to   { background: orange; }
		}
