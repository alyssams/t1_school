Floats

Elements are floated horizontally, this means that an element can only be floated left or right, not up or down.

A floated element will move as far to the left or right as it can. Usually this means all the way to the left or right of the containing element.


		float:left;
		float:right;

Clear

When we play with floating elements along with non-floating elements we can get into trouble. How so? Elements end up on top of each other.

So, what do we do. Let's clear the other elements on the page!


		clear:both;

Static Position

The position property specifies the type of positioning method used for an element (static, relative, absolute or fixed). By default, the position is static.

If you don't specify an element's positioning type, it defaults to static. This just means "where the element would normally go." If you don't tell an element how to position itself, it just plunks itself down in the document.

Relative Position

Relative positioning is more straightforward: it tells the element to move relative to where it would have landed if it just had the default static positioning.

offset properties

top:Specifies the top position of a positioned element

right:Specifies the right position of a positioned element

bottom:Specifies the bottom position of a positioned element

left:Specifies the left position of a positioned element

		
			top:20px;
			right:0;
			bottom:20px;
			left:0px;

Absolute Position

When an element is set to position: absolute, it's then positioned in relation to the first parent element it has that doesn't have position: static. If there's no such element, the element with position:absolute gets positioned relative to <html>.

Fixed Position

Fixed positioning anchors an element to the browser window. You can think of it as gluing the element to the screen. If you scroll up and down, the fixed element stays put even as other elements scroll past.
